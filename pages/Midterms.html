    • Cascading Style Sheets

    Language used to specify the presentation aspects (e.g layout and formatting) of structurally marked up documents.
    Developed by Hakon Wium Lie (HTML Cascading Style Sheets /CHSS) and Bert Bos (Stream-based Sheet Proposal / SSP).

    1. CSS Level 1 (CSS 1, W3C Recommendation, Dec 1996)

    2. CSS Level 2 (CSS 2, W3C Recommendation, May 1998)

    3. CSS Level 2 Revision 1 (CSS 2.1, W3C Recommendation, June 2011)

    4. CSS Level 3
      - modular approach to CSS development (as opposed to the monolithic specification of CSS 2.1)

    At - Rules

    • @charset - Defines the character set used by the style sheet.

    • @import - Tells the CSS engine to include an external style sheet.

    • @namespace - Tells the CSS engine that all its content must be considered prefixed with an XML namespace.

    • @media - A conditional group rule that will apply its content if the device meets the criteria of the condition defined using a media query.

    • @supports - Also a group rule that will apply its content if the browser meets the criteria of the given condition.

    • @page - Describes the aspect of layout changes that will be applied when printing the document.

    • @font-face - Describes the aspect of an external font to be downloaded.

    • @keyframes - - Describes the aspect of intermediate steps in a CSS animation.

    JavaScript

    JavaScript is the programming language of HTML and the Web.
    JavaScript was invented by Brendan Eich in 1995, and became an ECMA standard in 1997.
    Formerly called Mocha, then LiveScript, then Javascript.

      Terms:

    • Scripting Language, A programming language designed specifically for acting on an existing entity or system.

    • JavaScript engine, A program or interpreter that understands and executes JavaScript code.

    ECMAScript, The specification defined in ECMA-262 for creating a general purpose scripting language.

      Terms:

    • ECMA-262, This is a standard published by Ecma International. It contains the specification for a general purpose scripting language.

    Browser Object Model, Consists of the objects navigator, history, screen, location and document which are children of window.
    The Browser Object Model (BOM) allows JavaScript to "talk to" the browser.

      Terms:

    • Window Object
      Global variables are properties of the window object and global functions are methods of the window object.
      Example: window.document.getElementById("Header");

    DOM helps JavaScript to access and change all the elements of an HTML document.
    The DOM is a W3C (World Wide Web Consortium) standard.
    The W3C DOM standard is separated into 3 different parts:

    1. Core DOM - standard model for all document types
    2. XML DOM - standard model for XML documents
    3. HTML DOM - standard model for HTML documents

    Functionalities

    • Retrieve references to nodes (or node collection) in the DOM

      • getElementByID()

      • getElementsByTagName(), getElementsByClassName(), getElementsByName()

      • querySelector(), querySelectorAll()

    • Traverse the DOM tree (from a given node)

      • parentNode, parentElement

      • childNodes, children

      • firstChild, lastChild, nextSibling, previousSibling

      • firstElementChild, lastElementChild, nextElementSibling

    • Traverse the DOM tree (from a given node)

      • parentNode, parentElement

      • childNodes, children

      • firstChild, lastChild, nextSibling, previousSibling

      • firstElementChild, lastElementChild, nextElementSibling

    • construct/copy nodes:

      • createElement() - method creates the HTML element specified by tagName, or an HTMLUnknownElement if tagName isn't recognized.

      • createTextNode() - String that contains the data that to be put in the node

      • createAttribute() - Creates new attribute node and returns it

      • cloneNode() - Duplicates a node and returns it where the method is called

      • importNode() - Copies a node from other document and will insert it on the current document

    • manipulate the DOM tree:

      • appendChild() - Method adds a node to the end of the list of children of a specified parent node.

      • insertBefore() - Method inserts a node before the reference node as a child of a specified parent node.

      • replaceChild()-Method replaces one child node of the specified node with another.

      • removeChild() - Method removes a child node from the DOM. Returns removed node.

      • adoptChild() - Adopts a node. The node and its subtree is removed from the document it's in (if any), and its ownerDocument is changed to the current document.

    • miscellaneous attributes and methods:

      • nodeType- To ditingush kind of nodes, such that element, text and comments.

      • nodeName - Returns the name of the current node as a string

      • nodeValue - Returns or sets the value of the current node.

      • attributes - The namespace in which the target attribute and its associated values are defined.

      • id

      • tagName -Returns the tag name of the element on which it's called.

      • className - Gets and sets the value of the class attribute of the specified element.

      • classList- Returns a live DOMTokenList collection of the class attributes of the element.

      • ownerDocument - Read-only property returns the top-level document object for this node.

      • documentElement - Returns the Element that is the root element of the document .

      • hasChildNodes() - Method returns a Boolean value indicating whether the current Node has child nodes or not.

      • normalize() - ,ethod puts the specified node and all of its sub-tree into a "normalized" form.

    • manipulate attributes (for 'Element' nodes):

      • setAttribute()- Sets attribute to a element but if the element already exist its value updates

      • getAttribute() - Returns a element if not exist it is null

      • removeAttribute() - Removes an attribute from the specified element.

      • setAttributeNode() - Adds a new Attr node to the specified element.

      • getAttributeNode() - Returns the specified attribute of the specified element, as an Attr node.

      • removeAttributeNode() - Removes the specified attribute from the current element.

    • event handling:

      • addEventListener()-Sets up a function to be called whenever the specified event is delivered to the target.

      • removeEventListener() - The event listener to be removed is identified using a combination of the event type, the event listener function itself, and various optional options that may affect the matching process

      • dispatchEvent() - Invoking the affected EventListeners in the appropriate order.

    • style handling:

      • styleSheets -

      • ownerRule

      • cssRules

      • insertRule() - Method inserts a new CSS rule into the current style sheet, with some restrictions.

      • deleteRule() - Method removes a style rule from the current style sheet object.

      • getComputedStyle() - Method returns an object that reports the values of all CSS properties of an element after applying active stylesheets and resolving any basic computation those values may contain.
